# Project Roadmap

## Technical Development

1. Add seven-day weather forecast retrieval and normalization.
2. Let the agent evaluate a user-selected forecast date.
3. Expand the activity catalog with richer and more varied data.
4. Add controlled LLM-assisted activity generation.
5. Store user feedback and recommendation history.
6. Grow deterministic and LLM evaluation scenarios.
7. Prepare deployment only after the application workflow is complete.

## User Interface Notes

These items are intentionally postponed until the technical workflow is stable:

- Use consistent, natural Turkish throughout the application.
- Present a consumer-facing interface instead of a technical dashboard.
- Move evaluator results, scores, and agent traces into an optional developer
  details section.
- Show the seven-day forecast as a visual calendar/day selector similar to
  language-learning applications.
- Let users choose a location from a map.
- Improve activity cards, mobile layout, and user-friendly error messages.
